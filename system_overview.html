1. RFQ 詢價流程 業務 → 工程 → 採購 → BU → 客戶報價

業務收到客戶詢價後，建立 RFQ 單；工程排製程並派發供應商； 採購跟催；業務收齊報價後試算成本；BU 核准後發送客戶報價。

2. NPI 新產品開發流程 RFQ 客戶決定開發 → 工程 → BU 成案 → 採購議價 → 結案

客戶確認開發後，業務點選機種匯入 NPI；工程選定供應商並開立 ERP 模具請購單； 填寫 T1 試模日期；BU 核准成案；採購議價回填實際成本後結案，匯入 NAS。

4. PCN 變更流程（廠商主動） 工程建單 → 品保 SIP → 產線 SOP → BU 核准

廠商主動提出之製程變更，由工程建立，依序由品保填 SIP、產線主管填作業 SOP， 最後 BU 核准結案並產生 CC PDF。

5. ECN 變更流程（客戶主動 / 已量產設計） 兩支流程：技術類 + 商業類

客戶主動或已量產品設計變更。由 change_types JSON 決定走哪支： 「製程/設計/供應商變更」走技術類；「售價/成本變更」走商業類直送 BU。

6. QC 異常管理 NCR 流程 5 階段 + 三大處理路線 (A/B/C) + ECN 連結

IQC/IPQC/OQC 或產線發現異常時建單；品保下處理判斷（可多選）； 依路線執行（退貨／特採處理／橫向展開）；需長期改善則進入改善方案，必要時綁 ECN。

7. 訂單成本與售價流程 Alice 模組（HT_Production-system）
此模組由 Alice 主導，位於 HT_Production-system（目前為私有 repo）。 下方流程為依現有對接點推測之骨架，實際細節以 Alice repo 公開後為準。
推測流程圖（待 Alice 補完）
已知對接點（從 Kevin 模組看出去）
  • RFQ → 訂單：quoted_unit_price 為客戶報價單價，為訂單售價之來源
  • NPI → 訂單：mould_cost_final + bargain_data 為實際成本，供成本分析模組使用
  • ECN 商業類 → 訂單：售價/成本變更核准後，需同步至訂單模組 （目前以 PDF + Mail CC 通知，未直連系統）
  • QC NCR → 訂單：客戶端 Sorting/Rework 工時人力，可能納入成本
建議整合方向
  • Alice 提供訂單 API → Kevin 端可在 RFQ QUOTE_APPROVED 時自動建單
  • Kevin ECN 商業類核准 → POST 至 Alice 訂單成本/售價維護 endpoint
  • QC 客戶端工時人力 → 折算成本費用 push 至 Alice
技術架構
Kevin 模組（NPI 系統）

  • ERP 整合：pyodbc 直連 SQL Server (192.168.0.201)
  • 通知：LINE Messaging API + SMTP（urllib stdlib）
  • 啟動：uvicorn app.main:app --reload --port 8000
