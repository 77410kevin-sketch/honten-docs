已知對接點（從 Kevin 模組看出去）
  • RFQ → 訂單：quoted_unit_price 為客戶報價單價，為訂單售價之來源
  • NPI → 訂單：mould_cost_final + bargain_data 為實際成本，供成本分析模組使用
  • ECN 商業類 → 訂單：售價/成本變更核准後，需同步至訂單模組 （目前以 PDF + Mail CC 通知，未直連系統）
  • QC NCR → 訂單：客戶端 Sorting/Rework 工時人力，可能納入成本
建議整合方向
  • Alice 提供訂單 API → Kevin 端可在 RFQ QUOTE_APPROVED 時自動建單
  • Kevin ECN 商業類核准 → POST 至 Alice 訂單成本/售價維護 endpoint
  • QC 客戶端工時人力 → 折算成本費用 push 至 Alice
技術架構
Kevin 模組（NPI 系統）
  • 後端：FastAPI + SQLAlchemy async (aiosqlite)
  • 前端：Jinja2 + Bootstrap 5
  • DB：SQLite（dev）／可換 SQL Server（prod）
  • ERP 整合：pyodbc 直連 SQL Server (192.168.0.201)
  • 通知：LINE Messaging API + SMTP（urllib stdlib）
  • 啟動：uvicorn app.main:app --reload --port 8000
  • Repo： Honten-archive/feature/npi-module
Alice 模組（生產系統）
  • Repo： HT_Production-system （私有，待開放查閱）
  • 範圍：樣品需求單、訂單管理、成本分析、售價維護
  • 整合方式（建議）：REST API + 共用 SSO 帳號
